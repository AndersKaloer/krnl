37 \section a1 Before you start
38 
39 All initialization must be carried out after k_init and BEFORE k_start
40 
41 So no creation of semaphores, tasks, etc when the system is running.
42 
43 Why ? because we are dealing with embedded systems and once your system is configured it shall just run.
44 
45 I do also really advise not to use dynamic memory after k_start: no malloc and free when your system is running.
46 
47 If ... you need to do so please protect memory by a mutex (semaphore + k_wait and k_signal) or just encapsule malloc in DI() and EI().
48 And never never use and rely on free. You may end up with fragmented memory which on the long run is of no use.
49 
50 So the advise is to
51 
52 - create all tasks, semaphores etc
53 - allocate what is needed of memory, resources etc
54 - and then k_start

154 - preemptive scheduling

162  - krnl can be configures to use tmr 1,2 and for mega also 3,4,5 for running krnl tick
163  - For timer 0 you should take care of millis and it will require some modifications in arduino lib
164  - see krnl.h for implications (like
165 
166 - Accuracy
167  - 8 bit timers (0,2) 1 millisecond is 15.625 countdown on timer
168  - example 10 msec 156 instead of 156.25 so an error of 0.25/156.25 ~= 0.2%
169  - 16 bit timers count down is 1 millisecond for 62.5 count
170  - - example 10 msec ~ 625 countdown == precise :-)
171 
172 See in krnl.h for information like ...
173 
174 ... from http://blog.oscarliang.net/arduino-timer-and-interrupt-tutorial/
175 Timer0:
176 - Timer0 and 2 is a 8bit timer.
177 - In the Arduino world Timer0 is been used for the timer functions, like delay(), millis() and micros().
178 - If you change Timer0 registers, this may influence the Arduino timer function.
179 - So you should know what you are doing.
180 
181 Timer1:
182 - Timer1 is a 16bit timer.
183 - In the Arduino world the Servo library uses Timer1 on Arduino Uno (Timer5 on Arduino Mega).
184 
185 Timer2:
186 - Timer2 is a 8bit timer like Timer0.
187 - In the Arduino work the tone() function uses Timer2.
188 
189 Timer3, Timer4, Timer5: Timer 3,4,5 are only available on Arduino Mega boards.
190 - These timers are all 16bit timers.

206 - Timer0 - An 8 bit timer used by Arduino functions delay(), millis() and micros(). BEWARE
207 - Timer1 - A 16 bit timer used by the Servo() library
208 - Timer2 - An 8 bit timer used by the Tone() library
209 - Timer3,4,5 16 bits
210 
211 
212 ... from http://arduino-info.wikispaces.com/Timers-Arduino
213 
214 - Servo Library uses Timer1.
215 - - You can’t use PWM on Pin 9, 10 when you use the Servo Library on an Arduino.
216 - - For Arduino Mega it is a bit more difficult. The timer needed depends on the number of servos.
217 - - Each timer can handle 12 servos.
218 - - For the first 12 servos timer 5 will be used (losing PWM on Pin 44,45,46). -
219  - For 24 Servos timer 5 and 1 will be used (losing PWM on Pin 11,12,44,45,46)..
220 - - For 36 servos timer 5, 1 and 3 will be used (losing PWM on Pin 2,3,5,11,12,44,45,46)..
221 - - For 48 servos all 16bit timers 5,1,3 and 4 will be used (losing all PWM pins).
222 
223 - Pin 11 has shared functionality PWM and MOSI.
224 - - MOSI is needed for the SPI interface, You can’t use PWM on Pin 11 and the SPI interface at the same time on Arduino.
225 - - On the Arduino Mega the SPI pins are on different pins.
226 
227 - tone() function uses at least timer2.
228 - - You can’t use PWM on Pin 3,11 when you use the tone() function an Arduino and Pin 9,10 on Arduino Mega.
